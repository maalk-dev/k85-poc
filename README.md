# k85-poc

Proof of Concept for the thesis: *"Arkitekturstyrning av generativ AI i mjukvaruutveckling"*

## Overview

This repository is the technical infrastructure for a comparative evaluation of AI-generated code, with and without an architecture contract (`AGENTS.md`).

A single REST endpoint (`POST /orders`) is implemented on top of TypeScript, Express, Prisma and PostgreSQL. The `main` branch contains only infrastructure (dependencies, `tsconfig.json`, `openapi.yaml`, `prisma/schema.prisma`, `docker-compose.yml`). All application code under `src/` is generated by AI as part of the study.

The same prompt (`/implement-create-order`) is used in two scenarios:

- **Scenario A** — without `AGENTS.md`
- **Scenario B** — with `AGENTS.md`

The architecture contract is the only difference between scenarios. Each scenario lives on its own branch. Reviews are performed on dedicated `*-review` sub-branches using the `architecture-review` skill, with `AGENTS.md` excluded from the review context.

```
main
├── scenario-a            # generated code without architecture contract
│   └── scenario-a-review # architecture review
└── scenario-b            # generated code with architecture contract (AGENTS.md)
    └── scenario-b-review # architecture review
```

## Installation

### Prerequisites

- Node.js 24+
- pnpm 10+
- Docker (for PostgreSQL via `docker-compose`)
- VS Code with GitHub Copilot Chat

### Install

```bash
cp .env_example .env
pnpm install
docker compose up -d
pnpm db:generate
pnpm db:push
pnpm db:seed
```

## Running scenarios

```bash
# Scenario A
git checkout main && git checkout -b scenario-a
# (ensure AGENTS.md is NOT present)
# Run /implement-create-order in Copilot Chat
git add src && git commit -m "scenario A: AI-generated code"

# Scenario B
git checkout main && git checkout -b scenario-b
# (ensure AGENTS.md IS present at repo root)
# Run /implement-create-order in Copilot Chat
git add src && git commit -m "scenario B: AI-generated code"
```

## Review

Reviews are run on dedicated `*-review` sub-branches using the `architecture-review` skill. The analysis is written to `analysis/<branch-name>.md`.

```bash
# Review scenario A
git checkout scenario-a && git checkout -b scenario-a-review
# Run /architecture-review in Copilot Chat
git add analysis && git commit -m "scenario A: architecture review"

# Review scenario B
git checkout scenario-b && git checkout -b scenario-b-review
# Run /architecture-review in Copilot Chat
git add analysis && git commit -m "scenario B: architecture review"
```

## Scripts

| Script | Description |
|---|---|
| `pnpm dev` | Start API in watch mode |
| `pnpm test` | Run all tests (Vitest) |
| `pnpm lint` | Lint check (Biome) |
| `pnpm lint:fix` | Apply safe fixes |
| `pnpm db:generate` | Generate Prisma client |
| `pnpm db:push` | Push schema to database |
| `pnpm db:seed` | Seed database with test data |

## Project structure

```
openapi.yaml                                        # API specification (POST /orders)
prisma/schema.prisma                                # Database schema
prisma/seed.ts                                      # Test data (customers, products)
docker-compose.yml                                  # PostgreSQL
.github/prompts/implement-create-order.prompt.md    # Prompt (used in both scenarios)
AGENTS.md                                           # Architecture contract (scenario B only)
.github/skills/architecture-review/SKILL.md         # Review instructions (used on *-review branches)
analysis/<branch-name>.md                           # Review output (committed on *-review branches)
```
