# k85-poc

Proof-of-concept for the thesis *"Hur, och med vilken typ av effekt, kan ett arkitekturkontrakt — implementerat som en kontextfil (AGENTS.md) — påverka förekomsten av arkitekturavvikelser i AI-genererad kod?"*

## Overview

This repository is the technical infrastructure for a comparative evaluation of AI-generated code, with and without an architecture contract (`AGENTS.md`).

The development scenario is a single REST endpoint — `POST /orders` — implemented on top of TypeScript, Express, Prisma and PostgreSQL. Only infrastructure is committed on `main` (dependencies, `tsconfig.json`, `openapi.yaml`, `prisma/schema.prisma`, `docker-compose.yml`, seed data). All application code under `src/` is generated by AI as part of the experiment.

The experiment runs the same prompt (`/implement-create-order`) twice:

- **Scenario A** — without `AGENTS.md`
- **Scenario B** — with `AGENTS.md`

Each scenario lives on its own branch so the architecture contract is the only controlled difference. Review is performed on dedicated `*-review` sub-branches using the `architecture-review` skill, with `AGENTS.md` excluded from the review session.

```
main
├── scenario-a            ← run prompt, commit generated code
│   └── scenario-a-review ← run architecture-review skill
└── scenario-b            ← add AGENTS.md, run prompt, commit
    └── scenario-b-review ← run architecture-review skill
```

## Installation

### Prerequisites

- Node.js 24+
- pnpm 10+
- Docker (for PostgreSQL via `docker-compose`)
- VS Code with GitHub Copilot Chat (used to run the `/implement-create-order` prompt)

### Install

```bash
cp .env_example .env
pnpm install
docker compose up -d
pnpm db:generate
pnpm db:push
pnpm db:seed
```

Verify the scaffolding before running a scenario:

```bash
docker compose ps    # postgres should be running/healthy
pnpm db:seed         # idempotent — should succeed twice in a row
```

### Branch setup for a scenario

```bash
# Scenario A — no architecture contract
git checkout main
git checkout -b scenario-a
# (ensure AGENTS.md is NOT present)
# Run /implement-create-order in Copilot Chat
git add src && git commit -m "scenario A: AI-generated code"

# Scenario B — with architecture contract
git checkout main
git checkout -b scenario-b
# (ensure AGENTS.md is present at repo root)
# Run /implement-create-order in Copilot Chat
git add src && git commit -m "scenario B: AI-generated code"
```

## Running locally

Start the API in watch mode:

```bash
pnpm dev
```

Database utilities:

| Script | Description |
|---|---|
| `pnpm db:generate` | Generate Prisma client |
| `pnpm db:migrate`  | Run Prisma migrations |
| `pnpm db:push`     | Push schema to database |
| `pnpm db:seed`     | Seed database with test data |

## Testing

Unit tests run with Vitest. Note that on `main` there are no tests — `src/` is empty by design and is only populated when a scenario is run.

```bash
pnpm test                  # run all *.unit.test.ts once
pnpm test:specific         # watch mode
pnpm test:specific:logs    # watch mode, no log silencing
```

## Linting

Linting and formatting are handled by Biome:

```bash
pnpm lint              # check only
pnpm lint:fix          # apply safe fixes
pnpm lint:fix-unsafe   # apply unsafe fixes
```

## Project structure

```
openapi.yaml                                        # API contract (POST /orders)
prisma/schema.prisma                                # Database schema
prisma/seed.ts                                      # Test data (customers, products)
docker-compose.yml                                  # PostgreSQL
.github/prompts/implement-create-order.prompt.md    # Prompt used in both scenarios
AGENTS.md                                           # Architecture contract (scenario B only)
.github/skills/architecture-review/SKILL.md         # Review process (used on *-review branches)
```
